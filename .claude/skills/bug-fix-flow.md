# Bug 修复标准流程

## 流程（严格按顺序执行）

### 第一步：理解 Bug

- 阅读 Bug 报告，找到复现步骤
- 确认预期行为 vs 实际行为

### 第二步：写复现测试

- 写一个测试用例，精确复现这个 Bug
- **确认测试失败**（红色）才能继续
- 如果测试直接通过，说明 Bug 理解有误，回到第一步

### 第三步：修复代码

- 最小化修改范围
- 不要趁机重构或添加新功能

### 第四步：确认测试通过

- 运行复现测试：必须变绿
- 运行全部测试：不能引入新的失败

### 第五步：永久保留回归测试

- 这条测试不可删除
- 在注释中记录 Bug 背景

## 禁止行为

- 修复代码前不写复现测试
- 删除或修改回归测试
- 在修复 PR 里混入新功能
