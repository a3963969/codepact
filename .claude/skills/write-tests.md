# 如何写高质量测试

## 测试结构（AAA 模式）

每个测试用例遵循：
- **Arrange**：准备测试数据和环境
- **Act**：执行被测代码
- **Assert**：验证结果

## 必须覆盖的场景

1. **正常路径（Happy Path）**：标准输入，验证正确输出
2. **边界条件**：空值、null、极大值、极小值、空列表
3. **错误处理**：网络超时、权限不足、数据格式错误、并发冲突
4. **契约边界目录**：每个边界目录条目对应至少一个测试用例

## 测试质量标准

- 测试名称清晰表达"什么场景下期望什么结果"
- 每个测试只验证一件事
- 测试不依赖其他测试的执行顺序
- 所有外部依赖（数据库、API）使用 mock

## 回归测试规范

- Bug 修复产生的回归测试：在注释中记录对应的 Bug ID
- 回归测试永远不删除
- 命名格式：`test_regression_<bug_id>_<简短描述>`
