# 如何写 PR 描述

## 模板

```
## 关联契约
- 契约文件：`docs/contracts/YYYY-MM-DD_模块_功能.md`
- 契约状态：已确认 ✅

## 改动说明
<!-- 做了什么，为什么这么做（1-3 句话）-->

## 测试覆盖
- [ ] 正常路径
- [ ] 边界情况（对应契约边界目录）
- [ ] 错误处理

## 自检
- [ ] 所有测试通过
- [ ] 没有引入新的 lint 错误
- [ ] 核心逻辑是纯函数
- [ ] 覆盖率没有下降

## QA 审查
<!-- qa-review.yml 自动填入（需配置 ANTHROPIC_API_KEY Secret） -->
<!-- 请查看下方 Bot 评论中的 QA 审查结果，根据 severity 决定是否合并 -->
```

## 写作原则

- 改动说明面向 reviewer，解释"为什么"而不是"怎么做"
- Bug 修复 PR：简述 Bug 根因 + 修复思路
- 重构 PR：说明重构目标，确认"行为未变"
